🎯 Krok 1: Pierwsze wywołanie LLM


import requests, json

def call_llm(prompt, system="You are a helpful assistant."):
    response = requests.post(
        "http://localhost:11434/api/generate",
        json={
            "model": "gemma3:4b",
            "prompt": prompt,
            "system": system,
            "stream": False
        }
    )
    return json.loads(response.text)["response"]

# Test!
print(call_llm("What is machine learning? 2 sentences."))

📊 Co to jest LangGraph?

  • Graf = węzły + krawędzie
  • Węzły = kroki przetwarzania (klasyfikuj, szukaj, routuj)
  • Krawędzie = przepływ między krokami
  • Krawędzie warunkowe = „pewność > 80% → szybka ścieżka"
  • Stan = dane przepływające przez graf
classify → [confidence >= 0.8?] → YES → route
→ NO → search_kb → reclassify → route

Wyjaśnienie:
🔹 Graf = zbiór węzłów (kroków) i krawędzi (połączeń).
🔹 Krawędzie warunkowe = „jeśli confidence >= 0.8 → szybka ścieżka, inaczej → RAG".
🔹 Stan (State) = dane, które przepływają przez graf.

🏗️ Stan w LangGraph – TypedDict


from typing import TypedDict, Optional, List

class TicketState(TypedDict):
    text: str                    # Treść zgłoszenia
    category: Optional[str]      # Sprzęt / Sieć / Oprogramowanie / Konto
    confidence: float            # Pewność klasyfikacji (0.0 - 1.0)
    context: Optional[str]       # Kontekst z bazy wiedzy (RAG)
    route: Optional[str]         # Ścieżka: L1/L2/L3 support
    needs_review: bool           # Czy wymaga przeglądu człowieka?
    path: List[str]              # Logi: które węzły odwiedzone

🔑 Stan przepływa przez graf – każdy węzeł może go odczytać i zmodyfikować

Storytelling:
🔹 „TypedDict to jak formularz, który każdy pracownik na ścieżce wypełnia. Pierwszy wpisuje kategorię, drugi dodaje kontekst, trzeci decyduje o routingu. Formularz przepływa przez system — każdy uzupełnia swoją część."

Wyjaśnienie:
🔹 Pokaż kod i wyjaśnij każde pole: text = treść zgłoszenia, category = wynik klasyfikacji, confidence = jak pewny jest model, path = log ścieżki.
🔹 Kluczowy punkt: stan jest niezmienny — każdy węzeł ZWRACA nowy stan, nie modyfikuje starego.

🧠 Mity o AI – a jak jest naprawdę?

❌ Mit ✅ Rzeczywistość
„AI zastąpi wszystkich ludzi" AI wspiera ludzi – automatyzuje powtarzalne zadania
„Im większy model, tym lepszy" Phi-4 (3B) często wystarczy – dobieraj model do zadania
„AI zawsze wie, co mówi" Halucynuje – RAG i weryfikacja są konieczne
„Potrzebujesz chmury i drogiego GPU" Ollama na laptopie – za darmo, prywatnie, szybko
„Wystarczy napisać dobry prompt" Context Engineering: tools + memory + RAG + structured output

Jak poprowadzić:
🔹 Ten slajd porządkuje wiedzę z całego dnia. Odsłaniaj po jednym micie.
🔹 Zachęć uczestników do dodania swojego mitu: „Jaki mit o AI słyszycie najczęściej?"
🔹 Kluczowy nowy mit: „Wystarczy napisać dobry prompt" → Context Engineering.

Storytelling:
🔹 „Dwa dni temu wielu z Was myślało, że AI to czarna magia. Teraz wiecie, że to tokeny, attention, i HTTP requesty. A to, co wygląda na magię — to dobrze zaprojektowany kontekst."
